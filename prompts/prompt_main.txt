=== ПРОМПТ MAIN: РАЗБИВКА НА ЛОГИЧЕСКИЕ БЛОКИ ===
Проанализируй текст технического документа о производстве стали и раздели его на логические блоки.

ВАЖНО: Используй ТОЛЬКО следующие предопределенные типы блоков:
- document_info (базовая информация о документе)
- document_title (заголовок и метаданные документа)  
- chemical_composition (химический состав и таблицы)
- technical_requirements (технические требования и процессы)
- tables (дополнительные таблицы)

НЕ создавай блоки других типов! Если информация о стали не подходит под document_title, помести её в technical_requirements.

Если найден текст, который не подходит ни под один из этих типов, помести его в блок с типом "unknown".

Верни результат в формате JSON:
{
  "blocks": [
    {
      "block_type": "document_info",
      "content": "только номер документа, дата и список адресатов (БЕЗ заголовка документа)",
      "end_marker": "О производстве г/к сортового"
    },
    {
      "block_type": "document_title", 
      "content": "заголовок документа 'О производстве' и связанные с ним метаданные: вариант, код ОЭМК, код вида продукции, экспортное наименование, условный аналог, стандарт выплавки, № потребительских свойств",
      "start_marker": "О производстве г/к сортового",
      "end_marker": "ПРЕДЛАГАЮ"
    },
    {
      "block_type": "chemical_composition",
      "content": "текст блока с химическим составом и таблицами",
      "start_marker": "начало блока",
      "end_marker": "конец блока"
    },
    {
      "block_type": "technical_requirements",
      "content": "текст блока с техническими требованиями и процессами",
      "start_marker": "начало блока",
      "end_marker": "конец блока"
    },
    {
      "block_type": "tables",
      "content": "текст блока с дополнительными таблицами",
      "start_marker": "начало блока",
      "end_marker": "конец блока"
    },
    {
      "block_type": "unknown",
      "content": "текст, который не подходит под другие категории",
      "start_marker": "начало блока",
      "end_marker": "конец блока"
    }
  ]
}

ВАЖНО:
1. Разделяй текст на логические блоки по смыслу
2. Каждый блок должен содержать релевантный текст
3. Используй start_marker и end_marker для определения границ
4. Возвращай только валидный JSON без дополнительного текста
